¿Qué son los Lenguajes de Marcas?

Definición: Un lenguaje de marcas permite representar información mediante etiquetas que dan estructura, significado y formato a los datos.

Características importantes:

  • No son lenguajes de programación - No contienen instrucciones ni acciones directas
  • Organizan información - Proporcionan estructura uniforme
  • Facilitan procesamiento automático - Los programas pueden interpretar los datos

📊 Orientados a la presentación

Ejemplo: HTML
Para mostrar información en pantalla

💾 Orientados al almacenamiento

Ejemplo: XML
Para procesar e intercambiar datos

Ejemplo Práctico: HTML

Las etiquetas HTML indican al navegador cómo interpretar y mostrar el contenido:

<h1>El origen de la vida</h1>

🔍 ¿Qué hace esta etiqueta?

  • <h1> = Etiqueta de apertura
  • </h1> = Etiqueta de cierre
  • Indica que es un encabezado principal
  • El navegador lo muestra con tamaño de fuente mayor
  • Proporciona información semántica (significado)

💡 Ejercicio práctico: Crear una página web y probar la diferencia entre los encabezados <h1> al <h4>

Estructura de los Lenguajes de Marcas

📋 Reglas fundamentales (XML):

  • Etiqueta de apertura: <nombre>
  • Etiqueta de cierre: </nombre>
  • El contenido puede ser texto u otros elementos
  • Solo existe un elemento raíz
  • Los atributos van entrecomillados: atributo="valor"
  • Formato: texto plano

🎮 Ejemplo: videojuegos.xml

<videojuegos> <videojuego id="vg1"> <titulo>Videojuego multiplataforma</titulo> <desarrollador>Estudio XYZ</desarrollador> <plataforma>PC, PS5, Xbox</plataforma> </videojuego> </videojuegos>

Características Fundamentales

✅ Los lenguajes de marcas deben ser:

  • 🔄 Independientes: No dependen de plataforma hardware ni software específico
  • 📝 Texto plano: Almacenados como caracteres de texto simples
  • 🔧 Flexibles: Utilizables en diferentes contextos y aplicaciones
  • 🤝 Compatibles: Las marcas y el contenido están en el mismo documento
  • ⚡ Fáciles de procesar: Diseñados para procesamiento automático sencillo

🔍 HTML vs XML

¿Qué es un Dialecto?

📖 Definición de Dialecto:

Un dialecto es un lenguaje de marcas específico que se deriva de un metalenguaje (como XML o SGML). Es una variante especializada para un dominio o aplicación particular.

🏗️ Metalenguajes

  • SGML - Define reglas generales
  • XML - Metalenguaje simplificado

Establecen las reglas para crear lenguajes específicos

🎯 Dialectos (Lenguajes específicos)

  • HTML - Derivado de SGML
  • SVG - Dialecto de XML
  • MathML - Dialecto de XML
  • XHTML - Dialecto de XML

Implementan las reglas para casos específicos

🔗 Relación Metalenguaje → Dialecto:

XML (define estructura general) → SVG (gráficos vectoriales específicos)

SGML (estándar general) → HTML (páginas web específicas)

Ejemplo: Fichero XML

📄 biblioteca.xml

XML se usa para almacenar e intercambiar datos estructurados

<?xml version="1.0" encoding="UTF-8"?> <biblioteca> <libro id="libro1" genero="ficcion"> <titulo>El Quijote</titulo> <autor> <nombre>Miguel</nombre> <apellidos>de Cervantes</apellidos> </autor> <año_publicacion>1605</año_publicacion> <disponible>true</disponible> </libro> <libro id="libro2" genero="tecnico"> <titulo>Lenguajes de Marcas</titulo> <autor> <nombre>Ana</nombre> <apellidos>García López</apellidos> </autor> <año_publicacion>2023</año_publicacion> <disponible>false</disponible> </libro> </biblioteca>

🔍 Características del XML:

  • Declaración XML: Versión y codificación
  • Etiquetas personalizadas: biblioteca, libro, autor
  • Atributos: id="libro1", genero="ficcion"
  • Anidación: Estructura jerárquica clara

Ejemplos: Dialectos de XML

🎨 SVG - Gráficos Vectoriales

<?xml version="1.0"?> <svg xmlns="http://www.w3.org/2000/svg" width="200" height="200"> <circle cx="100" cy="100" r="80" fill="blue" stroke="red" stroke-width="3"/> <text x="100" y="110" text-anchor="middle" fill="white"> SVG </text> </svg>

🧮 MathML - Matemáticas

<?xml version="1.0"?> <math xmlns="http://www.w3.org/1998/Math/MathML"> <mrow> <mi>x</mi> <mo>=</mo> <mfrac> <mrow> <mo>-</mo><mi>b</mi> <mo>±</mo> <msqrt> <mrow> <msup><mi>b</mi><mn>2</mn></msup> <mo>-</mo> <mn>4</mn><mi>a</mi><mi>c</mi> </mrow> </msqrt> </mrow> <mrow><mn>2</mn><mi>a</mi></mrow> </mfrac> </mrow> </math>

🔑 Características de los dialectos:

  • SVG: Etiquetas específicas para gráficos (circle, text, path)
  • MathML: Elementos matemáticos (mfrac, msqrt, msup)
  • Namespace: xmlns define el vocabulario específico
  • Especialización: Cada dialecto optimizado para su dominio

  • JSON: Ideal para APIs y intercambio ligero de datos

HTML vs XHTML: ¿En qué se diferencian?

🔍 Dos enfoques para crear páginas web:

Aunque ambos sirven para crear páginas web, tienen diferencias importantes en cuanto a flexibilidad y rigurosidad

🌐 HTML (HyperText Markup Language)

  • Tolerante con errores
  • Flexible y fácil de usar
  • Los navegadores interpretan incluso con fallos
  • Puede generar problemas de compatibilidad
  • Errores invisibles en maquetación

📏 XHTML (eXtensible HyperText Markup Language)

  • Sintaxis estricta
  • Evolución de HTML adaptada a XML
  • Detiene carga si hay errores de sintaxis
  • Evita ambigüedades
  • Documentos más coherentes

🎯 Punto clave:

XHTML mantiene las mismas posibilidades de diseño que HTML, pero exige cumplir normas más estrictas para garantizar documentos bien formados y fácilmente procesables.

Diferencias Técnicas: HTML vs XHTML

📋 Requisitos de XHTML:

✅ Reglas obligatorias en XHTML:

  • Todas las etiquetas deben cerrarse correctamente
  • Las etiquetas deben estar bien anidadas (sin saltarse el orden de apertura y cierre)
  • Los atributos deben escribirse siempre con valor explícito
  • Sintaxis XML estricta (sensible a mayúsculas/minúsculas)

❌ HTML - Sintaxis flexible

<!-- HTML permite estos errores --> <p>Párrafo sin cerrar <br> <img src="foto.jpg"> <input type="text" checked> <!-- Mal anidamiento --> <p><strong>Texto</p></strong>

✅ XHTML - Sintaxis estricta

<!-- XHTML requiere sintaxis correcta --> <p>Párrafo correctamente cerrado</p> <br /> <img src="foto.jpg" alt="Foto" /> <input type="text" checked="checked" /> <!-- Anidamiento correcto --> <p><strong>Texto</strong></p>

⚠️ Comportamiento ante errores:

  • HTML: El navegador interpreta la página incluso con fallos
  • XHTML: Si existe un error de sintaxis, detiene la carga y muestra mensaje de error

Ejemplo Completo: Documento XHTML

📄 Ejemplo de documento XHTML válido:

Observa la declaración DOCTYPE, el namespace y la sintaxis estricta

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd"> <html xmlns="http://www.w3.org/1999/xhtml"> <head> <title>Mi primera página</title> </head> <body> <h1>Bienvenidos</h1> <p>Este es un párrafo</p> <br /> <img src="imagen.jpg" alt="Foto" /> </body> </html>

🔍 Elementos clave del ejemplo:

  • DOCTYPE: Especifica la versión XHTML 1.0 Strict
  • Namespace: xmlns="http://www.w3.org/1999/xhtml"
  • Etiquetas cerradas: <br />, <img ... />
  • Atributos completos: alt="Foto" (siempre con valor)

⚠️ IMPORTANTE:

Si se comete un error de sintaxis en este documento, el navegador mostrará un mensaje de error y detendrá la carga, evitando así ambigüedades.

Valor Actual de XHTML

🔄 Evolución hacia HTML5:

Aunque HTML5 ha sustituido a XHTML como estándar actual, conocer XHTML sigue siendo valioso

✅ Beneficios de conocer XHTML

  • Código limpio y ordenado
  • Documentos coherentes
  • Fácilmente procesables
  • Más seguros en validación
  • Buenas prácticas de programación

🎯 Aplicación práctica

  • Comprensión de sintaxis estricta
  • Importancia de la validación
  • Estándares de calidad en código
  • Base para XML y sus dialectos
  • Fundamentos de estructuras bien formadas

💡 Lección aprendida:

Las normas de XHTML nos enseñan la importancia de escribir código limpio, ordenado y correcto, principios que siguen siendo fundamentales en el desarrollo web moderno.

🚀 Legado en HTML5:

Muchas buenas prácticas de XHTML se han incorporado en el desarrollo moderno con HTML5, especialmente en frameworks y herramientas de validación.

1.5 Codificación de Archivos

📝 ¿Qué son los archivos de texto plano?

Archivos compuestos únicamente por caracteres de texto, a diferencia de los binarios que pueden contener imágenes, videos, audios, programas, etc.

⚠️ Problema histórico:

Incompatibilidad cuando se trabajaba con texto entre diferentes sistemas. Cada sistema usaba su propio código para representar caracteres.

🔤 Archivos de Texto

  • Solo caracteres de texto
  • Legibles directamente
  • Editables con cualquier editor
  • Portables entre sistemas
  • Seguros (no ocultan código)

💻 Archivos Binarios

  • Pueden contener cualquier dato
  • No legibles directamente
  • Requieren software específico
  • Más compactos
  • Procesamiento más rápido

🎯 Importancia para lenguajes de marcas:

Todos los lenguajes de marcas (HTML, XML, JSON) se almacenan en formato de texto plano, lo que garantiza su portabilidad y accesibilidad universal.

1.5.1 ASCII - American Standard Code

📅 Historia del ASCII (1967):

Creado por ANSI (American National Standards Institute) para resolver incompatibilidades entre sistemas. Utiliza el alfabeto inglés con caracteres latinos.

🔢 ASCII Original (7 bits):

  • 128 símbolos totales (2⁷ = 128)
  • Letras del alfabeto inglés (mayúsculas y minúsculas)
  • Números (0-9)
  • Caracteres de puntuación
  • Símbolos especiales
  • Símbolos de control (como salto de línea)
Ejemplos de códigos ASCII: A = 65 a = 97 0 = 48 B = 66 b = 98 1 = 49 Z = 90 z = 122 9 = 57 Espacio = 32 Enter = 13 Tab = 9

✅ Ventajas ASCII

  • Estándar universal
  • Compacto (7 bits)
  • Compatible con inglés
  • Base de otros sistemas

❌ Limitaciones ASCII

  • Solo alfabeto inglés
  • Sin caracteres especiales (ñ, ü, ç)
  • No sirve para otros idiomas
  • Necesita extensiones

ASCII Extendido (8 bits)

🔧 Solución para otros idiomas:

Se diseñaron códigos de 8 bits que añadían 128 símbolos más, manteniendo compatibilidad con ASCII original.

📊 Estructura ASCII Extendido:

  • Primeros 128 símbolos: Idénticos al ASCII original
  • Siguientes 128 símbolos: Caracteres específicos por región
  • Total: 256 símbolos (2⁸ = 256)
Código Región Caracteres especiales incluidos
ISO-8859-1 Europa Occidental ñ, ü, ç, ß, à, é, ì, ò, etc.
ISO-8859-2 Europa Central y del Este Ź, č, ě, ř, ş, ț, etc.
ISO-8859-3 Europa del Sur Ġ, Ħ, İ, Ĵ, etc.
ISO-8859-4 Europa del Norte ø, å, æ, ð, þ, etc.

🌍 Problema persistente:

Aunque ASCII extendido resolvió problemas regionales, seguía siendo insuficiente para un mundo global con múltiples idiomas simultáneos.

1.5.2 Unicode - Estándar Global

🌐 Objetivo de Unicode:

Aglutinar en una misma tabla de códigos todos los caracteres de cualquier lengua del planeta. Cada carácter ya no ocupa un solo byte.

🔢 Estructura Unicode:

  • Primeros 128: ASCII original (compatibilidad)
  • Siguientes 128: ISO-8859-1 (lenguas europeas occidentales)
  • Resto: Todos los idiomas del mundo (chino, árabe, japonés, etc.)
  • Gestión: Consorcio Unicode (unicode.org) + ISO

✅ Ventajas Unicode

  • Universal: Todos los idiomas
  • Compatible: Con ASCII y ISO-8859-1
  • Estándar: Apoyo industrial masivo
  • Extensible: Nuevos símbolos
  • Semántico: Cada símbolo tiene significado

⚖️ Consideraciones

  • Tamaño variable: No siempre 1 byte
  • Complejidad: Varios formatos (UTF-8, UTF-16, UTF-32)
  • Procesamiento: Más complejo que ASCII
  • Espacio: Puede ocupar más memoria

🎯 Resultado:

Unicode es el estándar definitivo para texto en la actualidad. Todos los sistemas modernos lo soportan.

Formatos Unicode: UTF-8, UTF-16, UTF-32

🔧 Tres formas de codificar Unicode:

Diferentes métodos para almacenar los caracteres Unicode, cada uno con ventajas específicas.

Formato Bytes por carácter Ventajas Uso típico
UTF-8 1-4 bytes (variable) Compatible con ASCII, muy extendido Web, archivos de texto
UTF-16 2-4 bytes (variable) Más simple que UTF-8 Windows internamente
UTF-32 4 bytes (fijo) Más simple de procesar Poco usado (desperdicia espacio)

📝 Ejemplo práctico UTF-8:

Texto: 取得cigüeña𐐝

  • 取得 (chino): 3 bytes cada uno = 6 bytes
  • c, i, g, e, a (ASCII): 1 byte cada uno = 5 bytes
  • ü, ñ (latinos): 2 bytes cada uno = 4 bytes
  • 𐐝 (persa antiguo): 4 bytes
  • Total: 19 bytes

🏆 UTF-8 - El ganador:

Aunque es más complejo de procesar, UTF-8 es el estándar claro para codificar texto. Todos los sistemas actuales tienen capacidad para codificar de esta forma.

🎯 Para lenguajes de marcas:

Los archivos de texto son ideales porque priorizan legibilidad, portabilidad y seguridad sobre velocidad de procesamiento.
